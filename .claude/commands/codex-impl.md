# Codex 구현 위임

다음 요구사항을 Codex CLI에 구현 작업으로 위임한다.

요구사항:

```
$ARGUMENTS
```

Codex에게 전달할 계약:

```text
다음 작업을 구현해줘.

요구사항:
$ARGUMENTS

역할:
- 너는 구현 담당자다.
- 파일 수정, 필요한 테스트 추가, 검증 실행까지 완료한다.

수정 원칙:
- 파일 수정은 요구사항과 직접 관련된 범위로 제한한다.
- 기존 사용자 변경을 revert하지 않는다.
- 요구사항과 무관한 refactor를 하지 않는다.
- dependency를 추가하지 않는다.
- git commit을 만들지 않는다.

TDD/테스트 기준:
- pure logic, validation, formatter, parser, helper, payload transform 변경이면 테스트 우선 흐름을 적용한다.
- TDD 대상이 아니라고 판단하면 종료 보고에 이유를 남긴다.
- 새 테스트를 추가하거나 삭제하면 관련 평가 문서 갱신 필요성을 확인한다.

검증:
- 완료 전 `pnpm verify:fast`를 실행한다.
- 실패하면 실패 원인, 해결한 항목, 남은 실패를 구분해 보고한다.

완료 보고:
- 변경 요약
- 검증 결과
- 테스트를 추가하지 않았다면 이유
- 남은 리스크 1줄

보고는 한국어로 작성한다. 코드 식별자, 파일 경로, 에러 메시지, 로그는 원문 그대로 유지한다.
```

실행:

```bash
codex exec "<위 계약과 요구사항을 포함한 프롬프트>"
```

Codex 완료 후 Claude는 `git diff`와 검증 결과를 확인하고, 계획과 어긋난 변경이 있으면 후속 수정을 요청한다.
