# Codex TDD 구현 위임

다음 요구사항을 TDD 흐름으로 Codex CLI에 위임한다.

요구사항:

```
$ARGUMENTS
```

Claude 진행 순서:

1. 성공 기준과 실패해야 하는 테스트 계획을 먼저 작성한다.
2. 사용자에게 테스트 계획 확인을 받는다.
3. 확인 후 Codex에 아래 계약으로 구현을 위임한다.
4. Codex 완료 후 `git diff`와 검증 결과를 리뷰한다.

Codex에게 전달할 계약:

```text
다음 작업을 TDD 흐름으로 구현해줘.

요구사항:
$ARGUMENTS

역할:
- 너는 구현 담당자다.
- 먼저 실패하는 테스트를 추가하거나 수정한다.
- 테스트 실패가 요구사항을 정확히 표현하는지 확인한 뒤 구현한다.
- 구현 후 작성한 테스트와 `pnpm verify:fast`가 통과하는지 검증한다.

수정 원칙:
- 파일 수정은 요구사항과 직접 관련된 범위로 제한한다.
- 기존 사용자 변경을 revert하지 않는다.
- 요구사항과 무관한 refactor를 하지 않는다.
- dependency를 추가하지 않는다.
- git commit을 만들지 않는다.

검증:
- 테스트 실패 확인 결과를 보고한다.
- 구현 후 테스트 통과 결과를 보고한다.
- 완료 전 `pnpm verify:fast`를 실행한다.

완료 보고:
- 변경 요약
- 추가/수정한 테스트
- 검증 결과
- 남은 리스크 1줄

보고는 한국어로 작성한다. 코드 식별자, 파일 경로, 에러 메시지, 로그는 원문 그대로 유지한다.
```

실행:

```bash
codex exec "<위 계약과 요구사항을 포함한 프롬프트>"
```
