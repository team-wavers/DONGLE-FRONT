# DONGLE 관리자 앱 내부 테스트 안내서

## 문서 목적
처음 사용하는 분도 관리자 앱을 직접 점검할 수 있도록 만든 안내서입니다.
아래 순서대로 진행하면 주요 기능을 빠짐없이 확인할 수 있습니다.

## 서비스 주소
admin.dev.dongle.wavers.kr

## 테스트 대상
- 총동연 관리자 화면
- 동아리 담당자 화면
- 동아리 신규 등록 화면

## 시작 전 준비
### 준비물
- 인터넷이 연결된 PC 또는 노트북
- 크롬 브라우저 권장
- 관리자 계정 1개
- 동아리 계정 1개

### 테스트 전 확인
- 브라우저 주소창에 admin.dev.dongle.wavers.kr 입력 시 로그인 화면이 보이는지 확인
- 이전 로그인 기록이 있다면 로그아웃 후 진행

## 테스트 진행 방식
### 1차 빠른 점검 (약 15분)
- 로그인 가능 여부
- 메뉴 이동 가능 여부
- 화면이 정상적으로 열리는지 확인

### 2차 기능 점검 (약 40분)
- 등록, 수정, 삭제가 실제로 되는지 확인
- 잘못 입력했을 때 안내 문구가 나오는지 확인

### 3차 최종 점검 (배포 전)
- 전체 흐름을 처음부터 끝까지 다시 점검
- 이전에 수정된 문제가 다시 생기지 않았는지 확인

## 공통 점검 항목 (모든 계정)
### 로그인 성공
- 올바른 아이디/비밀번호 입력
- 기대 결과: 로그인 성공 후 내 권한에 맞는 첫 화면으로 이동

### 로그인 실패
- 일부러 틀린 비밀번호 입력
- 기대 결과: 로그인 실패 안내가 보임

### 빈칸 제출
- 아이디 또는 비밀번호를 비우고 로그인 시도
- 기대 결과: 입력이 필요하다는 안내가 보임

### 권한별 화면 분리
- 동아리 계정으로 관리자 전용 메뉴 접근 시도
- 기대 결과: 접근이 제한되고 내 화면으로 이동

## 총동연 관리자 점검 항목
### 동아리 정보 관리
#### 메뉴 위치
동아리 관리

#### 확인할 기능
- 동아리 목록 보기
- 특정 동아리 상세 화면 열기
- 동아리 정보 수정
- 회장 정보 수정
- 동아리 삭제

#### 점검 포인트
- 필수 항목이 비어 있으면 저장되지 않아야 함
- 연락처 형식이 잘못되면 안내가 떠야 함
- 수정 후 다시 열었을 때 변경 내용이 유지되어야 함

### 동아리 등록 링크 발급
#### 메뉴 위치
동아리 등록

#### 확인할 기능
- 등록 링크 생성
- 링크 복사

#### 점검 포인트
- 생성 직후 링크가 보이는지
- 복사 버튼이 동작하는지
- 화면의 주의사항 안내가 잘 보이는지

### 활동보고서 관리
#### 메뉴 위치
활동보고서 관리

#### 확인할 기능
- 동아리별 보고서 목록 확인
- 보고서 상세 내용 확인

#### 점검 포인트
- 목록이 비어 있을 때 안내 문구가 자연스러운지
- 보고서 상세에서 제목, 작성일, 내용, 사진이 잘 보이는지

### 사용자 관리
#### 메뉴 위치
사용자 관리

#### 확인할 기능
- 사용자 목록 확인
- 사용자 추가
- 사용자 정보 수정
- 사용자 삭제

#### 점검 포인트
- 잘못된 형식 입력 시 안내가 떠야 함
- 추가/수정/삭제 후 목록이 즉시 반영되어야 함

### 메인 배너 관리
#### 메뉴 위치
배너 관리

#### 확인할 기능
- 배너 목록 보기
- 신규 배너 등록
- 기존 배너 수정
- 배너 삭제

#### 점검 포인트
- 이미지 없이 저장 시 안내가 떠야 함
- 기간이 잘못되면 저장되지 않아야 함
- 수정 후 목록에서 변경 내용이 확인되어야 함

## 동아리 담당자 점검 항목
### 동아리 정보 관리
#### 메뉴 위치
내 동아리 정보 관리

#### 확인할 기능
- 동아리 소개, 활동 정보, 연락처 등 수정
- 회장 정보 수정

#### 점검 포인트
- 저장 성공 안내가 보이는지
- 다시 진입했을 때 수정 내용이 유지되는지

### 활동보고서
#### 메뉴 위치
활동보고서

#### 확인할 기능
- 보고서 작성
- 보고서 상세 보기
- 보고서 수정
- 보고서 삭제

#### 점검 포인트
- 제목/내용이 너무 짧으면 안내가 떠야 함
- 사진 첨부 후 정상 저장되는지
- 삭제 후 목록에서 사라지는지

### 계정 정보 관리
#### 메뉴 위치
계정 정보 관리

#### 확인할 기능
- 아이디 변경
- 비밀번호 변경

#### 점검 포인트
- 현재 비밀번호를 틀리게 입력하면 변경되지 않아야 함
- 새 비밀번호 확인이 다르면 안내가 떠야 함

## 신규 동아리 등록 점검 항목
### 접속 경로
관리자가 발급한 등록 링크 접속

### 확인할 기능
- 신규 동아리 정보 입력 후 등록
- 등록 완료 화면 확인
- 발급된 임시 로그인 정보 복사

### 점검 포인트
- 링크가 유효하지 않으면 안내 화면이 보여야 함
- 등록 완료 후 다음 단계 안내가 명확해야 함

## 테스트 결과 작성 양식
### 기본 기록
- 테스트 일시
- 담당자
- 테스트 계정
- 점검 항목
- 실제 결과
- 판정 (정상/오류)
- 비고

### 오류 기록 시 포함 내용
- 어느 화면에서 발생했는지
- 어떤 순서로 눌렀는지
- 기대 결과와 실제 결과
- 재현 여부

## 최종 완료 기준
- 로그인, 메뉴 이동, 저장 기능이 모두 정상
- 등록, 수정, 삭제가 실제로 동작
- 오류 안내 문구가 사용자가 이해할 수 있게 표시
- 치명적인 문제 없이 주요 업무를 끝까지 수행 가능
