# Features

이 문서는 동글의 기능별 요구사항과 정책을 압축해서 정리한다.

## 일반 사용자 앱

### 메인 / 배너

- 사용자는 메인 화면에서 운영자가 등록한 배너를 볼 수 있다.
- 노출 가능한 배너만 사용자 앱에 표시한다.
- 배너 링크가 있으면 새 탭에서 이동한다.

### 동아리 목록

- 사용자는 동아리 목록을 볼 수 있다.
- 사용자는 검색어, 분과, 모집 상태로 목록을 좁힐 수 있다.
- 검색 결과가 없으면 조건에 맞는 empty-state를 보여준다.

### 동아리 상세

- 사용자는 동아리 기본 정보, 소개, 주요 활동, 모집 정보를 볼 수 있다.
- 사용자는 동아리 관련 외부 링크가 있으면 이동할 수 있다.
- 사용자는 동아리 활동보고서를 조회할 수 있다.
- 사용자는 공개 상태인 동아리 일정을 볼 수 있다.
- 동아리 상세에는 다가오는 일정을 먼저 보여주고, 지난 일정은 별도 영역으로 분리한다.
- 일정에는 제목, 유형, 일시, 장소, 설명, 외부 링크를 표시할 수 있다.

## 회장 앱

### 로그인

- 회장은 발급된 계정으로 로그인한다.
- 로그인 후 회장 권한으로 접근 가능한 화면만 사용한다.

### 동아리 정보 관리

- 회장은 담당 동아리의 소개, 주요 활동, 모집 상태, 모집 기간, 링크, 대표 이미지 정보를 수정할 수 있다.
- 모집중이면 모집 시작일과 마감일이 필요하다.
- 모집마감이면 모집 기간을 사용자 노출 정보에서 제거한다.

### 활동보고서 관리

- 회장은 담당 동아리의 활동보고서를 작성, 수정, 삭제할 수 있다.
- 활동보고서는 제목, 내용, 이미지를 포함할 수 있다.
- 내용은 빈 rich text 상태로 저장할 수 없다.

### 일정 관리

- 회장은 담당 동아리의 일정만 조회, 등록, 수정, 삭제할 수 있다.
- 일정 관리 메뉴는 목록 화면과 등록/수정 화면으로 구성한다.
- 목록 화면은 전체, 공개, 비공개, 다가오는 일정, 지난 일정 필터를 제공한다.
- 목록 항목에는 제목, 일시, 장소, 공개 여부, 일정 유형을 표시한다.
- 회장은 일정의 공개 여부를 설정할 수 있다.
- 공개 일정은 사용자 앱 동아리 상세에 노출될 수 있고, 비공개 일정은 회장과 관리자만 볼 수 있다.
- 일정 작성 필수값은 제목, 유형, 시작일시, 종료일시, 공개 여부다.
- 일정 작성 선택값은 장소, 설명, 외부 링크다.
- 일정 유형은 모집, 행사, 정기모임, 공지로 시작한다.

### 계정 정보 관리

- 회장은 본인 계정 정보를 확인하고 필요한 항목을 수정할 수 있다.

## 관리자 앱

### 로그인

- 관리자는 관리자 계정으로 로그인한다.
- 로그인 후 내부 복귀 경로만 허용한다.

### 동아리 관리

- 관리자는 동아리 목록을 조회하고 검색할 수 있다.
- 관리자는 동아리를 등록, 수정, 삭제할 수 있다.
- 관리자는 동아리 등록 URL을 발급할 수 있다.

### 일정 관리

- 관리자는 모든 동아리의 일정을 월간 캘린더에서 한 번에 볼 수 있다.
- 일정 관리 메뉴는 전체 일정 캘린더와 전체 일정 목록으로 구성한다.
- 캘린더는 날짜별 일정 개수를 파악할 수 있어야 한다.
- 캘린더 일정 표시는 동아리명과 일정 제목을 중심으로 짧게 보여준다.
- 관리자는 동아리명, 분과, 일정 유형, 공개 상태로 일정을 필터링할 수 있다.
- 관리자는 날짜를 선택해 해당 날짜의 일정 목록을 볼 수 있다.
- 관리자는 일정을 선택해 상세 정보를 확인할 수 있다.
- 관리자는 부적절하거나 노출하면 안 되는 일정의 공개 상태를 변경할 수 있다.
- 초기 정책상 관리자는 일정 내용을 직접 수정하지 않고, 공개 상태 변경 또는 삭제로 운영 제어한다.

### 사용자 관리

- 관리자는 사용자 목록을 조회하고 검색할 수 있다.
- 관리자는 관리자 계정을 생성할 수 있다.
- 관리자는 사용자 정보를 수정할 수 있다.

### 배너 관리

- 관리자는 사용자 앱 메인에 노출할 배너를 등록, 수정, 삭제할 수 있다.
- 배너는 이미지, 노출 기간, 사용 여부, 링크 정보를 가진다.
- 사용자 앱에는 사용 중이고 노출 기간에 포함된 배너만 표시한다.

## 공통 정책

- 전화번호는 휴대폰 번호 형식만 허용한다.
- 주요 텍스트 입력값은 저장 전에 trim 정규화한다.
- 외부 URL은 허용 가능한 형태로 정규화하거나 저장하지 않는다.
- 권한이 없는 사용자는 해당 역할의 화면과 액션에 접근할 수 없다.
- 사용자에게 노출되는 일정은 공개 상태인 일정만 포함한다.
- 일정은 시작일시가 종료일시보다 늦을 수 없다.
- 삭제된 일정은 사용자 앱과 관리자 기본 목록에 노출하지 않는다.
