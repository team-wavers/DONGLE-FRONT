# Decisions

이 문서는 제품 기획 결정과 보류 항목을 짧게 기록한다.

## 확정

- 제품 기획 문서는 `docs/product/README.md`, `features.md`, `decisions.md` 3개로 유지한다.
- 역할, 권한, 화면 흐름, 주요 정책은 기능 단위로 `features.md`에 함께 적는다.
- API 요청/응답 상세는 중복 작성하지 않고 `API.md`를 기준으로 삼는다.
- 테스트 성공 기준과 제품 기획은 분리한다. 검증 계약은 `docs/evals/*`에 둔다.
- 배포와 운영 전략은 제품 기획 문서에 넣지 않고 `docs/deployment-strategy.md`에 둔다.
- 동아리 일정은 회장이 직접 등록, 수정, 삭제한다.
- 최고관리자는 회장들이 등록한 전체 일정을 월간 캘린더와 목록에서 확인한다.
- 최고관리자는 초기 정책상 일정 내용을 직접 수정하지 않고, 공개 상태 변경 또는 삭제로 사용자 노출을 제어한다.
- 일반 사용자는 동아리 상세에서 해당 동아리의 공개 일정을 확인하고, 사용자 앱 전체 일정 페이지에서 공개 동아리 일정과 총동연 공통 일정을 함께 확인한다.
- 사용자용 전체 일정은 헤더 전역 진입점에서 접근할 수 있게 한다.
- 일정 유형은 모집, 행사, 정기모임 3개로 운영한다.

## 보류

- 화면별 상세 와이어프레임 문서는 아직 만들지 않는다.
- 권한 매트릭스 문서는 별도로 분리하지 않는다.
- 릴리즈 로드맵 문서는 확정된 일정이 생기기 전까지 만들지 않는다.
